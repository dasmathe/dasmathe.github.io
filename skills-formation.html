Intégrer des solutions dans un environnement de développement, de test ou de production

4

Pour chaque nouvelle fonctionnalité ajoutée à BeeApee, j'ai suivi le processus suivant :

  • Gestion de version avec GitLab : Chaque nouvelle fonctionnalité est développée sur une branche Git basée sur la branche master, permettant une gestion incrémentale et isolée des fonctionnalités.
  • Tests : Avant tout déploiement, des tests simulant différents scénarios d'utilisation doit être fait. Les tests doit être d'abord effectués localement. Puis une fois validés, les fonctionnalités sont déployées sur le serveur de préproduction pour des tests supplémentaires.
  • Déploiement : Pour mettre à jour le serveur de préproduction, il faut utiliser GitLab et ensuite compilé les builds localement avant de les transférer en ftp sur le serveur.
  • Validation par plusieurs parties prenantes : Avant la livraison finale, les fonctionnalités développées sont validées par plusieurs membres de l'équipe, assurant ainsi la qualité et la conformité aux attentes du client.

Accompagner le management de projet informatique

3

L'agence Genesii fonctionne en méthode agile avec un système de lot de fonctionnalités à développer pour chaque projet. Chaque lundi matin, toute l'équipe se réunit pour discuter du planning hebdomadaire. Lors de cette réunion, nous faisons un tour de table pour partager les informations de veille technologique ou sectorielle pertinentes. En fonction des besoins, des points d'avancement supplémentaires sont organisés au fil des jours pour traiter des retours clients ou des bugs inattendues. Pour chaque nouvelle fonctionnalité, nous faisont une nouvelle branche GitLab basée sur la branche master. Grâce à Gitlab, nous pouvons déployer facilement les nouvelles fonctionnalités sur le serveur de pré-production pour des tests supplémentaires et après leurs validations nous les mettons en production.
